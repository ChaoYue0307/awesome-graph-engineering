The discipline

Loops made agents programmable.
Graphs make organizations programmable.

One agent running a loop can find work, act, verify, and retry. Graph engineering is the layer above: designing many agents as one system — stable roles with real boundaries (the org graph), ephemeral task structures generated per job (the work graph), and evidence gates on every edge that matters. Nine design layers cover the practice:

🎭

1 · Roles

Who exists and what each owns: capability boundaries, dedicated context, model-tier staffing — frontier orchestrators, cheaper workers.

🕸️

2 · Topology

How roles are arranged: pipelines, orchestrator–worker, councils and debate, hierarchies, meshes — shape matched to task.

🤝

3 · Handoffs

What crosses each edge, in what shape: contracts, structured outputs, artifacts vs messages, MCP and A2A.

🧩

4 · Work graphs

The structure this job needs right now: decomposition, DAG generation, routers, fan-out, dynamic spawning.

🗄️

5 · State

What nodes share and what stays isolated: blackboards, artifact stores, checkpoints, worktrees, write conflicts.

🛡️

6 · Gates

Where evidence decides: judge nodes, quorum votes, anti-groupthink checks, tests and CI as oracles between stages.

🚨

7 · Reliability

When a node fails mid-graph: cascades, retries, budgets, resume and replay, kill switches, human escalation.

📈

8 · Observability & cost

Seeing and affording the graph: trace views, token economics per node, critical-path latency, cost per outcome.

🧬

9 · Evolution

Graphs that redesign themselves: dynamic agent orgs, topology search, learned communication structures.
